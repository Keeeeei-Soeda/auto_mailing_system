## 実装方針
- Python で Gmail API を用いてメールを作成する。
- MIME は `multipart` とし、上記本文（プレーンテキスト）＋PDF添付（`application/pdf`）を組み立てる。
- 認証は既存の OAuth クライアント／トークン（`k.soeda@medi-canvas.com` の `gmail.compose` または `gmail.send` スコープ）を再利用する。無ければ新規に取得する。
- **既定の挙動は「下書き（draft）作成」とすること。** 送信前に内容と添付を目視確認したいため、いきなり `send` はしない。
- 確認後に送信する場合に備え、`--send` フラグ等で `send` にも切り替えられるようにしておく。

## 実行手順
1. 添付PDFを所定パスに配置する。
2. スクリプトを実行し、上記内容で Gmail に**下書き**を作成する。
3. Gmail の下書きを開き、宛先・件名・本文・添付を確認する。
4. 問題なければ下書きから送信、またはスクリプトを送信モードで再実行する。

## 完了条件
- 指定の宛先・件名・本文・添付PDFを含む下書きが Gmail 上に作成されている。
- 添付PDFが破損なく開ける。
