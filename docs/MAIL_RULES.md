# メール送信ルールブック

このファイルは、メール作成・送信時に **必ず守るルール** をまとめたものです。
別のチャット（セッション）に移動しても、このルールを読めば同じ品質・同じ手順で送信できるようにすることが目的です。

> 新しいチャットを始めたら、まずこの `MAIL_RULES.md` と `SETUP.md` を読むこと。

---

## 0. 送信の仕組み（前提）

- 送信は主に `send-mail.py`（または `send-mail.mjs`）を使い、GAS Webアプリ経由で送信する（詳細は `SETUP.md`）。
- 送信元は常に `k.soeda@medi-canvas.com` 固定。
- 認証情報は `.env`（`GAS_EXEC_URL` / `GAS_TOKEN`）。`.env` は絶対にコミットしない。
- 添付は `--attach`、既存スレッド返信は `--thread-id`。

### コマンド形式

```bash
# テスト・確認用（テストアドレスへ実際に送る）
python3 send-mail.py --to "pharnewton@gmail.com" --subject "件名" --body "本文..."

# 本送信（添付・CC・スレッド返信の例）
python3 send-mail.py \
  --to "宛先" \
  --cc "cc@example.com" \
  --subject "件名" \
  --body "本文..." \
  --attach "attachments/xxx.pdf" \
  --thread-id "（必要なときのみ）"
```

## 1. 送信フロー（厳守）

1. ユーザーが「宛先・相手・用件」を伝える。
2. エージェントが件名・本文を整える（下記フォーマットルールに準拠）。
3. **必ず先にテストメールを `pharnewton@gmail.com` に送る**（本番宛先には送らない）。
   - このシステムでは `--dry-run` ではなく、テストアドレスへの実送信で内容確認を行う運用。
4. ユーザーがテストメールを確認し、修正があれば反映して再度テスト送信する。
5. ユーザーが「送って」「OK」と明言したら、**本番宛先**へ本送信する。
6. 「送って」「OK」と言われるまで、本番宛先への送信は絶対にしない。
7. 実送信は取り消せないため、宛先のタイプミスは特に注意して確認する。

---

## 2. 本文フォーマットルール

### 2-1. 宛名部分（冒頭）

- 「団体名・役職・氏名」は **空行を入れず、連続した行** で書く。
  - 例：
    ```
    NPO法人線維筋痛症友の会
    理事長 兼 北海道支部 支部長
    久保田様
    ```
  - ❌ 各行の間に空行を入れない（1行空けない）。

### 2-2. 宛名の後

- 宛名（「〜様」）の後は **1行空けて** 本文（あいさつ）に入る。

### 2-3. 本文の段落

- 段落と段落の間は **1行空ける**（読みやすさのため）。
- あいさつ（「いつもお世話になっております。」など）の後も1行空ける。

### 2-4. 署名（末尾）

- 結びの言葉（「何卒よろしくお願い申し上げます。」）の後に1行空ける。
- 会社名と氏名は **空行を入れず連続** で書く。
  - 例：
    ```
    株式会社メディキャンバス
    代表取締役　副田　渓
    ```

### 2-5. 全体テンプレート

```
（団体名）
（役職）
（氏名）様

いつもお世話になっております。

株式会社メディキャンバスの副田です。
（本文1段落目…）

（本文2段落目…）

（結びの言葉）

株式会社メディキャンバス
代表取締役　副田　渓
```

---

## 3. 件名ルール

- 簡潔で用件が伝わるものにする。
- ユーザーから件名の指定・修正があれば、それを優先する。

---

## 4. 文体・トーン

- 丁寧なビジネス文面（敬語）。
- 相手の負担に配慮した一文を入れる（例：「ご多忙のことと存じますので、もしお時間が難しいようでしたら、どうぞお気になさらないでください。」）。
- 送信者の名乗りは「株式会社メディキャンバスの副田です。」。

---

## 5. 過去の修正履歴から得た学び（追記していく）

- 宛名の役職で「副理事長 兼 ○○支部 支部長」のような「兼」表記は、指定があれば役職ごとに改行して分ける。
- 「二段改行（行間に空行）」と「空行なしの連続行」はユーザーの指定で使い分ける。デフォルトの宛名は **空行なしの連続行**。

---

## 6. チェックリスト（送信前に毎回確認）

- [ ] 宛名の3行（団体・役職・氏名）は空行なしで連続しているか
- [ ] 宛名の後・段落間・結びの後は適切に1行空いているか
- [ ] 署名（会社名・氏名）は空行なしで連続しているか
- [ ] 件名はユーザー指定どおりか
- [ ] まずテストアドレス（`pharnewton@gmail.com`）へ送ったか
- [ ] 本番宛先のスペル・ドメインは正しいか（To / CC の両方）
- [ ] 添付がある場合、ファイル名・内容は正しいか
- [ ] スレッド返信の場合、`--thread-id` を指定したか
- [ ] ユーザーの「送って／OK」を得てから本送信したか
